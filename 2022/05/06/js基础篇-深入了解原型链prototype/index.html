# js基础篇：深入了解原型链prototype

JavaScript 是一种基于原型的语言，通过一条完整的原型链来共享对象的属性和方法，本章节主要对继承和原型链解释

# prototype

先简单创建个对象

const obj = {};
// {}

我们知道这是一个空对象，但是尝试下面操作:

console.log(obj.toString())
// [object Object]
console.log(obj.valueOf())
// {}

这些都是我们在编程中经常用到得操作，不为陌生。那么有没有思考过，这些对象方法是如何来的？下面一步一步来解析。

const obj = {}; 其实是 const obj = new Object(); 的语法糖，使用对象构造函数来创建一个对象.每个对象都有一个名为 [[Prototype]]（我们可以叫它为原型），因为这是一个内部属性，不能直接访问，所以js提供了一个 proto 属性，通过__proto__来访问：

console.log(obj.__proto__)
// {constructor: ƒ, __defineGetter__: ƒ, __defineSetter__: ƒ, hasOwnProperty: ƒ, __lookupGetter__: ƒ, …}

obj的__proto__指向构造函数Object的原型，当你试图访问一个对象的属性或方法时，JavaScript 会首先搜索对象本身，如果没有找到，它会搜索对象的 原型.

// Object.getPrototypeOf() 方法返回指定对象的原型
Object.getPrototypeOf(toString);
// ƒ () { [native code] }

这就解释了为什么obj会有toString方法。

我来看看另外一种对象类型：数组，您知道它们有许多内置方法，例如pop()和push()。创建新数组时可以访问这些方法的原因是因为您创建的任何数组都可以访问Array.prototype。

我们尝试创建它：

const arr = [];

他的原理跟obj差不多，可以编写为数组构造函数创建：const arr = new Array()。

我们再来看看它的__propto__属性：

console.log(arr.__proto__)
// [constructor: ƒ, concat: ƒ, pop: ƒ, push: ƒ, …]

我们可以看到Array.prototype的属性，都被arr的__proto__所继承了，当我们在使用arr.push()时查询到arr上没有该属性时，就会往arr.__proto__上查找，也就是Array.prototype。

这样看是不是简单明了？我们在来点复杂的。

const arr2 = [1, 2, 3];
arr2.toString();
// 1,2,3

js做了什么？先是发现自身没有toString，然后会在arr2._proto_ 上找，而arr2._proto_ 只有 pop、push等，没有toString，于是去 arr2._proto_._proto_ 上找， arr2._proto_._proto_ 就是 Object.prototype！

这里需要理解的是：Array是构造函数，构造函数本事也是一个对象，如上述，对象会怎样？没错，凡是对象，就会有一个__proto__属性指向它的构造函数prototype。这就是传说中的原型链了，它看起来应该是：arr2 -> Array -> Object。

如果在查询对象 原型 后仍然没有找到匹配项，JavaScript 将检查链接对象的 原型，并继续搜索，直到到达原型链的末尾。

在原型链的末端是Object.prototype. 所有对象都继承 的属性和方法Object。任何超出链末尾的搜索尝试都会导致null.

// Object.getPrototypeOf() 方法返回指定对象的原型
console.log(obj.__proto__.__proto__;);
// null
